This guide explains how to connect RisingWave to a PostgreSQL database to ingest data changes in real time. RisingWave's PostgreSQL CDC connector is compatible with any PostgreSQL-compliant database that supports logical replication. It has been tested with PostgreSQL versions 10 through 16.

### 1. Set up your PostgreSQL database

Before connecting to RisingWave, you must enable logical replication on your upstream database. The specific steps depend on your provider.

Select your platform below for detailed setup instructions:

*   ➡️ **[Set up a self-hosted PostgreSQL database](./self-hosted)**
*   ➡️ **[Set up AWS RDS or Aurora PostgreSQL](./aws-rds)**
*   ➡️ **[Set up a Neon serverless PostgreSQL database](./neon)**
*   ➡️ **[Set up a Supabase project](./supabase)**

Once setup is complete, return to this guide to connect RisingWave.

### 2. Connect RisingWave to PostgreSQL

RisingWave uses the `CREATE SOURCE` statement to establish a shared connection to your database and the `CREATE TABLE` statement to ingest data from specific tables.

#### `CREATE SOURCE` (shared source)

Use `CREATE SOURCE` to define the connection parameters for your PostgreSQL database. This creates a reusable source that multiple tables can draw from.

```sql
CREATE SOURCE pg_source WITH (
    connector='postgres-cdc',
    hostname='your_host',
    port='5432',
    username='your_user',
    password='your_password',
    database.name='your_database',
    schema.name='public'
);
```

#### `CREATE TABLE ... FROM source`

Use `CREATE TABLE` to ingest a specific upstream table. You must define the schema and specify a primary key that matches the upstream table's primary key.

```sql
CREATE TABLE my_table (
    id INT PRIMARY KEY,
    name VARCHAR
)
FROM pg_source TABLE 'public.my_upstream_table';
```

### Automatically map upstream table schema

Instead of defining columns individually, you can use `*` to ingest all columns from the source table. The table in RisingWave will be created with the same column names and types as the upstream table, with the upstream primary key automatically inferred.

```sql
CREATE TABLE my_table (*) FROM pg_source TABLE 'public.my_upstream_table';
```

### Extract metadata from sources

The `INCLUDE` clause allows you to add metadata fields from the CDC event to your table as new columns. For details, see [Extract metadata from sources](/ingestion/advanced/extract-metadata-from-sources).
