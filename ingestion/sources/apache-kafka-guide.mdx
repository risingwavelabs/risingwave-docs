This guide shows you the fastest way to connect RisingWave to your Apache Kafka broker.  You can choose to either create a *source* or a *table*:

*   **`CREATE SOURCE`:**  Creates a connection to Kafka but *does not* store the data in RisingWave.  This is useful for quickly inspecting data or for streaming-only use cases.
*   **`CREATE TABLE`:** Creates a connection to Kafka *and* stores the ingested data within RisingWave. This is useful for historical analysis and data warehousing.

For detailed configuration options and advanced settings, see [Apache Kafka Configuration Options](/path/to/kafka-config-options.md).  For information on supported data formats, see [Apache Kafka Data Formats](/path/to/kafka-data-formats.md).

## Prerequisites

*   A running Apache Kafka cluster.
*   The broker address(es) (`ip:port`) of your Kafka cluster.
*   The topic name you want to consume from.
*   Network access from your RisingWave instance to your Kafka brokers.

## Basic Connection

### Using `CREATE SOURCE` (Non-Persistent)

This example creates a connection to Kafka *without* storing the data:

```sql
CREATE SOURCE IF NOT EXISTS my_kafka_source (
    column1 data_type,  -- Replace with your actual column names and types
    column2 data_type,
    ...
)
WITH (
   connector='kafka',
   topic='your_topic_name',
   properties.bootstrap.server='your_broker_address:9092' -- e.g., 'localhost:9092' or 'broker1:9092,broker2:9092'
) FORMAT PLAIN ENCODE JSON;
```

### Using `CREATE TABLE` (Persistent)

This example creates a connection to Kafka and stores the data in RisingWave:

```sql
CREATE TABLE IF NOT EXISTS my_kafka_table (
    column1 data_type,  -- Replace with your actual column names and types
    column2 data_type,
    ...
)
WITH (
   connector='kafka',
   topic='your_topic_name',
   properties.bootstrap.server='your_broker_address:9092' -- e.g., 'localhost:9092' or 'broker1:9092,broker2:9092'
) FORMAT PLAIN ENCODE JSON;
```

## Essential parameters

The following parameters are required for a basic connection:

|Parameter|Description|
|---|---|
|`connector` |Must be set to 'kafka'.	|
|`topic`|The name of the Kafka topic you want to consume from. Example: `topic='my_topic'`|
|`properties.bootstrap.server`|The address(es) of your Kafka broker(s). Use a comma-separated list for multiple brokers. Example:'broker1:9092,broker2:9092'|
|FORMAT	|Must be set to `PLAIN`|
|ENCODE	|The encoding of the data. This example uses JSON. Other options available in Apache Kafka Data Formats.|

## Verify the connection

After running the `CREATE SOURCE` or `CREATE TABLE` command, you can verify the connection:

### Querying a source

```sql
SELECT * FROM my_kafka_source;
```
### Querying a table

```sql
SELECT * FROM my_kafka_table;
```

If data is flowing from your Kafka topic, you should see results.
