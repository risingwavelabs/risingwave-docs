RisingWave supports multiple deployment modes, each designed for different use cases and environments. This guide helps you choose the right option quickly.

## How to choose (quick checklist)

- **Just want to try RisingWave locally?** Use **Standalone mode**.
- **Need production reliability with minimal ops?** Use **[RisingWave Cloud](/deploy/risingwave-cloud)**.

## Quick comparison

| Deployment option | Time to deploy | Production use? | Best for | Key prerequisites | Documentation |
|---|---:|---|---|---|---|
| **Standalone mode** (Script, Docker, or Homebrew) | < 5 minutes | No | Learning, development, quick experiments | Install via script, Docker, or Homebrew | [Standalone mode](/operate/rw-standalone-mode) |
| **RisingWave Cloud** | < 5 minutes | Yes | Managed production deployments | [RisingWave Cloud account](/deploy/risingwave-cloud#prerequisites) | [RisingWave Cloud](/deploy/risingwave-cloud) |
| **Docker Compose (testing only)** | < 5 minutes | No | Local testing with bundled services | [Docker Desktop and local resources](/deploy/risingwave-docker-compose#prerequisites) | [Docker Compose](/deploy/risingwave-docker-compose) |
| **Kubernetes (Helm)** | ~30 minutes | Yes | Self-managed production deployments (simpler) | [Kubernetes 1.24+, Helm 3.7+, cluster resources](/deploy/risingwave-k8s-helm#prerequisites) | [Kubernetes (Helm)](/deploy/risingwave-k8s-helm) |
| **Kubernetes (Operator)** | ~30 minutes | Yes | Self-managed production deployments (more automation) | [kubectl, Docker, psql, cluster resources](/deploy/risingwave-kubernetes#prerequisites) | [Kubernetes (Operator)](/deploy/risingwave-kubernetes) |

## Recommendations

### Quick start and development

**Recommended: Standalone mode**

Best for learning, development, and quick testing. It runs as a single process and gets you started in minutes.

[Learn more →](/operate/rw-standalone-mode)

### Production environments

**Recommended: RisingWave Cloud** (managed) or **Kubernetes** (self-managed)

- **RisingWave Cloud**: Fully managed. Best when you want production reliability without infrastructure management.
- **Kubernetes**: Self-managed. Best when you need full control and already operate Kubernetes in production.

[RisingWave Cloud →](/deploy/risingwave-cloud) · [Kubernetes (Helm) →](/deploy/risingwave-k8s-helm) · [Kubernetes (Operator) →](/deploy/risingwave-kubernetes)

### Testing and single-machine deployment

**Recommended: Docker Compose**

Easy setup for local testing with multiple services. Not recommended for production.

[Learn more →](/deploy/risingwave-docker-compose)
