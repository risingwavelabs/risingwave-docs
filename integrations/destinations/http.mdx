Use the HTTP sink connector to deliver sink records to an HTTP service or webhook endpoint with `POST` requests.

## Syntax

```sql
CREATE SINK [ IF NOT EXISTS ] sink_name
[FROM sink_from | AS select_query]
WITH (
    connector = 'http',
    url = '<http_endpoint_url>',
    type = 'append-only',
    content_type = '<mime_type>',
    header.<header_name> = '<header_value>'
)
;
```

## Parameters

| Parameter | Description |
| :-- | :-- |
| `url` | **Required.** The HTTP endpoint URL that receives `POST` requests. |
| `type` | **Required.** Must be `append-only`. |
| `content_type` | **Optional.** The `Content-Type` header value. If not set, RisingWave uses `text/plain` for `varchar` payloads and `application/json` for `jsonb` payloads. |
| `header.<header_name>` | **Optional.** Custom request headers. For example, `header.x_test = 'rw-http-sink'`. |

## Behavior and constraints

- The sink sends one `POST` request per inserted row.
- The sink input must have exactly one column.
- The sink column type must be `varchar` or `jsonb`.
- `NULL` payload values are skipped.
- `auto.schema.change = 'true'` is not supported.

For upstream tables with updates and deletes, set `ignore_delete = 'true'` so only inserted rows are delivered:

```sql Example
CREATE TABLE t_ignore_delete_src (payload varchar);

CREATE SINK s_ignore_delete FROM t_ignore_delete_src WITH (
    connector = 'http',
    url = 'http://localhost:18081',
    type = 'append-only',
    ignore_delete = 'true'
);
```

## Example

```sql Example
CREATE TABLE t_varchar (payload varchar);

CREATE SINK s_varchar FROM t_varchar WITH (
    connector = 'http',
    url = 'http://localhost:18081',
    type = 'append-only',
    force_append_only = 'true',
    header.x_test = 'rw-http-sink'
);

INSERT INTO t_varchar VALUES ('hello world');
INSERT INTO t_varchar VALUES ('{"key":"value"}');
```
