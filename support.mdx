Need help with RisingWave? This page will guide you to the right support channel based on your question or issue.

## Quick decision guide

Use this table to find the right support channel for your needs:

| Question Type | Recommended Channel | Details |
|--------------|---------------------|---------|
| **General questions** | [#ask-ai on Slack](https://risingwave-community.slack.com/archives/C06R8DA3DGT) | AI-powered answers based on docs and community discussions |
| **Technical questions** | [Community Slack](https://www.risingwave.com/slack) | Ask in topic-specific channels (see below) |
| **Bug reports** | [GitHub Issues](https://github.com/risingwavelabs/risingwave/issues/new/choose) | File detailed bug reports with reproducible steps |
| **Feature requests** | [GitHub Issues](https://github.com/risingwavelabs/risingwave/issues/new/choose) | Propose new features or enhancements |
| **Cloud support** | [cloud-support@risingwave-labs.com](mailto:cloud-support@risingwave-labs.com) | For RisingWave Cloud customers |
| **Sales inquiries** | [sales@risingwave-labs.com](mailto:sales@risingwave-labs.com) | Pricing, licensing, enterprise plans |
| **Security issues** | [GitHub Security Advisories](https://github.com/risingwavelabs/risingwave/security/advisories) | Report security vulnerabilities privately |

## Support channels

### Community Slack workspace

Join the [RisingWave Community Slack](https://www.risingwave.com/slack) for real-time help and discussions.

**Popular channels by topic:**

- **#ask-ai** — AI assistant for quick answers (recommended first stop)
- **#troubleshooting** — Help with errors, performance issues, and debugging
- **#general** — General discussions about RisingWave
- **#connectors** — Questions about sources, sinks, and integrations
- **#sql** — SQL syntax, query optimization, and functionality
- **#deployment** — Kubernetes, cloud deployments, and infrastructure
- **#performance** — Performance tuning and optimization
- **#cloud** — RisingWave Cloud-specific questions

<Tip>
**Pro tip:** Search the Slack history before posting. Many common questions have already been answered!
</Tip>

### GitHub Issues

For bugs, feature requests, and detailed technical discussions, use [GitHub Issues](https://github.com/risingwavelabs/risingwave/issues).

**When filing an issue, include:**

- A clear summary of the problem or feature request
- Steps to reproduce (for bugs)
- Expected vs. actual behavior
- RisingWave version and deployment environment
- Relevant logs or error messages

See [File an issue](/troubleshoot/overview#file-an-issue) for more details.

## Escalation paths

### For urgent production issues

1. **RisingWave Cloud customers:** Email [cloud-support@risingwave-labs.com](mailto:cloud-support@risingwave-labs.com) with "URGENT" in the subject line
2. **Open-source users:** Post in [#troubleshooting on Slack](https://risingwave-community.slack.com/archives/C0AE8A81LE6) with "[URGENT]" in your message
3. **Security vulnerabilities:** Use [GitHub Security Advisories](https://github.com/risingwavelabs/risingwave/security/advisories) to report privately

### Service Level Agreements (SLA)

RisingWave Cloud customers on **Premium** and **Enterprise** plans have access to:
- Guaranteed response times
- Dedicated support channels
- Quarterly success reviews (Enterprise only)

See [RisingWave Cloud pricing](/cloud/pricing) for details on support tiers.
