---
title: "SHOW SUBSCRIPTIONS"
description: "Use the `SHOW SUBSCRIPTIONS` command to list all subscriptions in a schema."
---

## Syntax

```sql
SHOW SUBSCRIPTIONS [ FROM schema_name ] [ LIKE_expression ];
```

## Parameters

| Parameter or clause | Description                                                                                                                                                                               |
| :------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| _schema\_name_      | The schema of the subscriptions to be listed. The default schema is public.                                                                                                               |
| LIKE\_expression    | Filters the output based on names by applying pattern matching. See details in [LIKE pattern matching expressions](/sql/functions/string#like-pattern-matching-expressions). |

## Example

```sql
SHOW SUBSCRIPTIONS;
```

```bash
        Name
---------------------
 public.my_sub
 public.order_events_sub
 public.metrics_stream_sub
(3 rows)
```

<Note>
Starting from v2.7.0, the `Name` column in the output includes the schema name as a prefix (e.g., `public.subscription_name`). This helps distinguish subscriptions with the same name in different schemas when querying across multiple schemas in the search path.
</Note>
