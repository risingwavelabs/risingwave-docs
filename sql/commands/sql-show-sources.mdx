---
title: "SHOW SOURCES"
description: "Use the `SHOW SOURCES` command to show existing sources."
---

## Syntax

```bash
SHOW SOURCES [ FROM schema_name ] [ LIKE_expression ];
```

## Parameters

| Parameter or clause | Description                                                                                                                                                                               |
| :------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| _schema\_name_      | The schema of the sources to be listed. If not specified, sources from all schemas in the current [schema search path](https://www.postgresql.org/docs/current/ddl-schemas.html#DDL-SCHEMAS-PATH) will be listed. |
| LIKE\_expression    | Filters the output based on names by applying pattern matching. See details in [LIKE pattern matching expressions](/sql/functions/string#like-pattern-matching-expressions). |

## Examples

```sql
SHOW SOURCES;
```

```bash
        Name
---------------------
public.nics_metrics
public.tcp_metrics
public.twitter_events
(3 rows)
```

<Note>
Starting from v2.7.0, the `name` column in the output includes the schema name as a prefix (e.g., `public.source_name`). This helps distinguish sources with the same name in different schemas when querying across multiple schemas in the search path.
</Note>
