---
title: "DROP SUBSCRIPTION"
description: "Use the `DROP SUBSCRIPTION` command to remove a subscription if you no longer need to monitor data changes."
---

## Syntax

```sql
DROP SUBSCRIPTION [ IF EXISTS ] [schema_name.]subscription_name;
```

## Parameters

| Parameter or clause | Description |
| :------------------ | :---------- |
| **IF EXISTS** | Do not return an error if the specified subscription does not exist. |
| _schema\_name_ | The schema of the subscription that you want to remove. You can use [SHOW SCHEMAS](/sql/commands/sql-show-schemas) to get a list of all available schemas. If you don't specify a schema, the specified subscription in the default schema `public` will be removed. |
| _subscription\_name_ | The name of the subscription to remove. |

## Examples

This statement removes the subscription `sub_users` in the default schema `public`:

```sql
DROP SUBSCRIPTION sub_users;
```

This statement removes the subscription `sub_events` in the schema `app_schema`:

```sql
DROP SUBSCRIPTION IF EXISTS app_schema.sub_events;
```

## Notes

- Dropping a subscription will remove all associated subscription cursors.
- Any client that is actively consuming from this subscription will be disconnected.
